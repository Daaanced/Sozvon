UX/UI спецификация — это документ, описывающий пользовательский опыт (UX) и пользовательский интерфейс (UI) продукта. Он включает в себя логику работы, структуру, функциональность и визуальное оформление, чтобы сделать взаимодействие с цифровым продуктом максимально удобным, понятным и приятным для пользователя. Такая спецификация нужна для того, чтобы вся команда понимала, как должен работать и выглядеть итоговый продукт.

- Проектирование **пользовательского пути (User Journey Mapping)** для понимания сценариев взаимодействия пользователя с продуктом.
- Проработка **логики и сценариев** работы интерфейса, чтобы достичь цели пользователя.

User Journey Mapping

Пользователь начинает работу с первого окна клиента - окна логина, содержащего поля для ввода логина и пароля. Есть отдельная кнопка ***регистрация*** для создания новой учетной записи.
После создания новой учётной записи он попадает на главную страницу профиля ~~со следующим содержимым:~~
~~***Сбоку слева***:~~
~~Чаты с пользователями~~
В центральном поле он находит по логину или UID пользователя, может начать с ним общение через текстовый или голосовой чат.
Во время голосового чата пользователь может изменить громкость своего микрофона, а так же изменить громкость собеседника, кликнув правой кнопкой мыши по его иконке.
В настройках пользователь может изменить способ ввода и вывода информации.
